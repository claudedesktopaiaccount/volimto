# Database Runbook

## Canonical Sources

- App schema source of truth: `src/lib/db/schema.ts`.
- Migration source of truth: `drizzle/` generated by Drizzle Kit for PostgreSQL.
- Neon project: `volimto` (`damp-shadow-59441201`).
- Canonical data branch ID: `br-silent-violet-alrxatck`.
- Target branch name: `staging`; if Neon still shows `codex-setup`, rename is pending.
- Legacy `production` branch was observed empty on 2026-05-28; do not use it as a data source until it is deliberately promoted or repointed.
- Neon-managed `neon_auth` schema is outside the app Drizzle schema unless the app starts managing it directly.

## Migration Workflow

1. Change `src/lib/db/schema.ts`.
2. Generate SQL with `npm run db:generate`.
3. Review generated SQL before applying it.
4. Apply shared Neon changes with `npm run db:migrate` against the intended branch.
5. Run `npx drizzle-kit check` before merging DB changes.

Use `npm run db:push` only for disposable local/dev databases. Do not use `db:push` against shared Neon branches because it bypasses reviewed migration history.

## Baseline

The active `drizzle/` directory is a squashed PostgreSQL baseline generated from `src/lib/db/schema.ts`.

Historical SQLite/D1/mixed SQL files were archived under `docs/db/legacy-migrations/drizzle-pre-baseline/`. They are preserved for audit only and must not be used as active Drizzle migrations.

For an existing Neon branch that already has the baseline schema, create `drizzle.__drizzle_migrations` and insert the baseline marker instead of running `0000_baseline_postgres.sql` over existing tables.

## Data Patches And Seeds

- Core party seed: `npm run db:seed`.
- Strana vidieka/Huliak faction patch: `npm run db:patch:strana-vidieka`.
- The old one-off SQL patch is archived at `docs/db/legacy-migrations/manual-sql/seed-strana-vidieka.sql`.
- New manual SQL patches must either become typed idempotent scripts in `scripts/` or be followed by a Drizzle migration and archived.

## Neon Branch Rename

Rename the canonical branch from `codex-setup` to `staging` with a trusted authenticated Neon CLI or Neon Console session:

```bash
neonctl branches rename codex-setup staging --project-id damp-shadow-59441201
```

After rename, verify the branch ID stayed `br-silent-violet-alrxatck`. The local sandbox could not run `neonctl` safely because it would need to download and execute an npm package outside the workspace.
